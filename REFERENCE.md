## Установка

1. Убедитесь, что у вас установлен **[Node.js](https://nodejs.org/en/download) версии 18 или выше**;
2. Установите утилиту глобально, чтобы использовать команду `paper-compose` из любой директории:

```bash
npm i -g paper-compose
```

## Обновление и удаление

### Обновление до последней версии

```bash
npm update -g paper-compose
```

### Удаление

```bash
npm uninstall -g paper-compose
```

## Базовое использование

### Простой пример

Создайте Markdown файл с вашей работой:

```markdown
# Введение

Это первый параграф моей работы.

## Раздел 1

Текст раздела.
```

Конвертируйте в Word:

```bash
paper-compose my_paper.md
```

Результат сохранится в файл `my_paper.docx` в той же директории.

### Синтаксис команды

```bash
paper-compose <file>.md
```

**Параметры:**
- `<file>.md` — путь к Markdown файлу для обработки (обязательный аргумент)

**Выходной файл:**
- Автоматически создаётся файл с тем же именем, но расширением `.docx`
- Находится в той же директории, что и исходный `.md` файл

## Структура Markdown файла

Типичный Markdown файл для paper-compose должен содержать:

### 1. Frontmatter (опционально)

В начале файла может быть YAML-конфигурация, ограниченная строками `---`:

```yaml
---
title: "Моя курсовая работа"
author: "Иван Петров"
description: "Описание работы"
toc: true
---
```

### 2. Содержимое документа

После frontmatter идёт основной текст с поддерживаемым синтаксисом Markdown.

### Полная структура

```markdown
---
title: "Название"
author: "Автор"
description: "Описание"
toc: true
---

# Заголовок уровня 1

## Заголовок уровня 2

### Заголовок уровня 3

Обычный параграф текста.

- Маркированный список
- Второй пункт

1. Нумерованный список
2. Второй пункт

```code block here```

| Колонка 1 | Колонка 2 |
|-----------|-----------|
| Данные    | Данные    |

![Alt текст](image.png)
```

## Поддерживаемый синтаксис

### Заголовки

Поддерживаются **три уровня** заголовков:

```markdown
# Заголовок уровня 1
## Заголовок уровня 2
### Заголовок уровня 3
```

**Особенности:**
- Заголовок 1 уровня автоматически добавляет разрыв страницы перед собой
- Все заголовки форматируются по ГОСТу с увеличенным размером шрифта и жирным начертанием
- Заголовки автоматически добавляются в оглавление

### Форматирование текста

```markdown
**жирный текст**
*курсивный текст*
***жирный и курсивный***
`код в строке`
```

**Типографирование происходит автоматически:**
- Умные кавычки («»)
- Правильные тире (—)
- Правильные многоточие (…)
- Другие типографические символы

### Списки

#### Маркированный список

```markdown
- Первый пункт
- Второй пункт
- Третий пункт

* Можно также использовать звёздочки
* Второй пункт
```

#### Нумерованный список

```markdown
1. Первый пункт
2. Второй пункт
3. Третий пункт
```

**Особенности:**
- Маркированные списки используют булеты (•)
- Нумерованные списки используют десятичную нумерацию с точкой

### Листинги кода

Код в блоках автоматически нумеруется как листинги:

````markdown
```
def hello_world():
    print("Hello, World!")
```
````

**Результат в DOCX:**
- Перед кодом добавляется строка "Листинг N"
- Код форматируется шрифтом Courier New, размер 10pt
- Вокруг кода рисуется граница
- Каждый блок кода нумеруется последовательно

### Таблицы

GFM-таблицы поддерживаются полностью:

```markdown
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-----------|-----------|-----------|
| Данные 1  | Данные 2  | Данные 3  |
| Данные 4  | Данные 5  | Данные 6  |
```

**Особенности:**
- Таблицы автоматически нумеруются как "Таблица N"
- Перед таблицей добавляется строка с номером
- Таблица занимает 100% ширины страницы
- Все ячейки имеют границы
- Каждая таблица нумеруется последовательно

### Изображения

```markdown
![Подпись к изображению](path/to/image.png)
![Alt текст](https://example.com/image.jpg)
```

**Особенности:**
- Изображения центрируются на странице
- Alt-текст или URL становится подписью под изображением
- Поддерживаются локальные пути и URL (http/https)
- Размер изображения автоматически рассчитывается

**Поддерживаемые форматы:**
- JPEG
- PNG
- GIF
- BMP
- SVG

### Разделители

```markdown
---
```

Создаёт разрыв страницы.

## Frontmatter конфигурация

Frontmatter — это YAML-блок в начале Markdown файла, ограниченный строками `---`. Он должен быть самым первым элементом файла.

### Синтаксис

```yaml
---
title: "Название работы"
author: "Автор"
description: "Краткое описание"
toc: true
---
```

### Доступные поля

#### `title`
- **Тип:** строка
- **Описание:** Название документа. Сохраняется в метаданные DOCX файла.
- **Пример:** `title: "Курсовая работа по программированию"`

#### `author`
- **Тип:** строка
- **Описание:** Автор документа. Сохраняется в метаданные DOCX файла.
- **Пример:** `author: "Иван Петров"`

#### `description`
- **Тип:** строка
- **Описание:** Описание документа. Сохраняется в метаданные DOCX файла.
- **Пример:** `description: "Анализ алгоритмов сортировки"`

#### `toc`
- **Тип:** булево значение (true/false)
- **По умолчанию:** true
- **Описание:** Включить или отключить автоматическое оглавление. Если `false`, оглавление не будет добавлено в документ.
- **Пример:** `toc: false`

### Примеры frontmatter

#### Полная конфигурация

```yaml
---
title: "Анализ методов оптимизации"
author: "Петров И.В."
description: "Курсовая работа на тему сравнения методов оптимизации"
toc: true
---
```

#### Без frontmatter

Если frontmatter не указан, утилита использует значения по умолчанию:
- Оглавление будет включено
- Метаданные будут пусты

```markdown
# Первый заголовок

Текст работы...
```
